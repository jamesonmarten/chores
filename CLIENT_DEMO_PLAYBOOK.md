## 1. Pre-Demo Checklist (5 minutes)

1. Open the demo page: https://chores.devcabin.tech/demo
2. Confirm both tabs work:
- Parent View
- Kid View
3. Click 2-3 chores in Kid View to verify points/progress updates.
4. Start one timer-based chore to confirm timer UX.
5. Test one voice/video helper button if visible.
6. Keep two browser tabs ready:
- Tab A: /demo (for guided walkthrough)
- Tab B: /app (to show production app)

Optional but recommended:
- Use a private/incognito window to avoid stale cache.
- Keep mobile viewport ready (or iPad) to show touch UX.

## 3. 10-Minute Discovery Demo (Client Call)

### Minute 0-2: Problem Framing

1. Ask:
- "What chore system do you use today?"
- "Where does it break down most often?"
2. Confirm pain points:
- Inconsistent follow-through
- Parent reminders/nagging
- No clear reward loop

### Minute 2-5: Parent Workflow

1. Open Parent View in /demo or /app.
2. Show:
- Family dashboard stats
- Kid progress cards
- Parent visibility into completion
3. Explain business value:
- Less manual tracking
- Faster daily check-ins
- Better consistency

### Minute 5-8: Kid Workflow

1. Switch to Kid View.
2. Select a kid profile.
3. Demonstrate:
- Tap-to-complete chore flow
- Point gain and progress movement
- Timers/helpers where present
4. Emphasize behavior design:
- Clear task list
- Instant reinforcement
- Repeatable routine loop

### Minute 8-10: Commercial + Next Step

1. Pricing line:
"It starts with a free trial, then $6/month for full Pro access."
2. Clarify setup:
"No complex install required for web use."
3. Ask for close:
- "Would you like to start with the trial now?"
- "Or schedule a 20-minute onboarding session?"

## 5. Objection Handling (Client-Friendly)

"My kids won’t use it"
- "The kid view is built for fast tap interaction and instant feedback, so adoption is usually much easier than paper charts."

"We already use a checklist app"
- "Most checklist apps track tasks, but this adds kid-specific motivation and parent visibility in one flow."

"Setup sounds hard"
- "For web use, setup is lightweight. We can walk through it in under 20 minutes."

"Is this only for one child?"
- "No, it is built for multi-kid households with per-kid progress visibility."

## 6. Client Onboarding Steps (After They Say Yes)

1. Share app link: https://chores.devcabin.tech/app
2. Create household profile and confirm parent mode access.
3. Add each child profile.
4. Add initial chores for each child (start with 3-5 each).
5. Set point values by effort level.
6. Enable any helper content needed (timer/video/voice prompts).
7. Run first family walkthrough in Kid View.
8. Define a weekly reward rule (example: 1 treat after all daily chores).
9. Schedule a 7-day check-in call.
10. Review adoption and adjust chore list difficulty.

Recommended rollout pattern:
- Week 1: consistency over complexity.
- Week 2: optimize points and rewards.
- Week 3: expand routines and helper content.

---

## 7. Success Metrics To Track

Track these in follow-up calls:

- Daily completion rate per child
- Number of parent reminders required per day
- Weekly streak consistency
- Family satisfaction (simple 1-5 score)

Target outcomes in first 30 days:
- Higher completion consistency
- Fewer parent reminders
- Faster morning/evening routine execution
